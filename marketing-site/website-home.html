FAQ

Frequently Asked Questions

Everything you need to know about MindWave.

Who can use MindWave?

MindWave is built for three roles — Faculty (who create and run sessions), Students (who participate and track progress), and HOD/Admin (who manage the institution). Access is role-gated and controlled by your institution's admin.

How does the live session work?

Faculty launch a session which generates a session code. Students enter the code in their dashboard and join instantly. Responses are tracked live, with a real-time leaderboard, anti-cheat monitoring, and a results report available the moment the session ends.

What does the anti-cheat system detect?

The system monitors tab switches, copy-paste attempts, browser focus loss, and keyboard shortcuts. Each violation is logged and flagged in the faculty dashboard so you can review suspicious activity after the session.

Is student data stored securely?

Yes. MindWave is self-hosted, meaning your institution's data stays within your own environment. All sessions are encrypted and access is strictly role-based — students can only see their own data.

Can faculty see individual student performance?

Yes. The analytics dashboard shows per-student scores, time taken, correct/incorrect answers, and participation trends across all sessions. HODs can see aggregate data across all faculty sections in their department.
